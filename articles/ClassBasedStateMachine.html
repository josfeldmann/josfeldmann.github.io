C#/Unity Class Based State Machine

State Machines are some of the first coding patterns you may learn about in game development. They are a way of determining how a certain peice of code may execute based on the "State" that its in. The most common example of a state machine is storing a state in a state enum variable and then every frame having a switch statement or a chain of if else statements that determine what code should be executed based on the saved state. Here is an example of a unity class that uses this pattern to do that. 

public class State {

    public virtual void Enter(StateMachine obj) { }

    public virtual void Exit(StateMachine obj) { }

    public virtual void Update(StateMachine obj) { }

}
public class StateMachine {

	public State currentState;
	public T target;

	public StateMachine(State startState, T obj) {
		currentState = startState;
		target = obj;
		currentState.Enter(this);
	}

	public void ChangeState(State state) {
		currentState.Exit(this);
		currentState = state;
		state.Enter(this);
	}

	public void Update() {
		currentState.Update(this);
	}
			
	public void FixedUpdate() {
		currentState.FixedUpdate(this);
	}
}
